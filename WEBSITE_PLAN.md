# Garvit Singh Personal Website Plan

## Purpose

This website is the official personal website and portfolio of Garvit Singh. It will showcase my projects, open-source work, writing, resume, and software engineering journey.

The first goal is to make the website exist with strong content and clear structure. After that, the design, animations, and visual polish will be improved step by step.

## Website Positioning

Garvit Singh is a Software Product Engineering student focused on backend APIs, full-stack products, open-source contributions, and real-world software systems.

The website should communicate that I am serious, consistent, project-driven, and open to remote software engineering internships.

## Primary Audience

* Recruiters
* Founders
* Internship hiring teams
* Senior developers
* Open-source maintainers
* College/project reviewers
* People discovering my work through GitHub or LinkedIn

## Main Call to Action

The main visitor action should be:

* View Projects
* Download Resume

## Design Direction

The website should feel:

* Clean
* Serious
* Fast
* Technical
* Recruiter-friendly
* Product-minded
* Easy to scan

Avoid:

* Too many animations
* Heavy 3D effects
* Particle backgrounds
* Overdesigned cards
* Generic portfolio templates
* Fake-looking skill percentages

## Content Rule

The website should be honest and interview-safe.

Every claim on the website should be something I can explain clearly in an interview.

## AI Usage Rule

AI can be used to improve speed and quality, but the website should not be blindly vibe-coded.

AI can help with:

* Drafting content
* Improving wording
* Creating issue descriptions
* Suggesting UI structure
* Generating component boilerplate
* Reviewing grammar
* Improving accessibility

AI should not be used to publish content without manual review.

Every AI-generated change must be checked manually before merging.
